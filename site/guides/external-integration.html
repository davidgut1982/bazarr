External Integration (Jellyfin & VLSub) v2.2

Bazarr+ v2.2 (Synapse) ships an OpenSubtitles-compatible REST endpoint that lets external clients query your Bazarr+ instance as a federated subtitle service. Two first-party clients ship alongside it: a Jellyfin plugin and a VLC extension.

What is External Integration

Bazarr+ already searches and downloads subtitles from your configured providers (OpenSubtitles.com, OpenSubtitles.org via the scraper, Subscene, Addic7ed, and many more) according to your priority order, your hashes, your language profiles, and your filters.

External Integration exposes that pipeline over a stable REST API at /compat/*. Any client that speaks the OpenSubtitles wire format can talk to Bazarr+ and use your provider configuration. No provider keys leave your server, no rate limit gets shared with the public OpenSubtitles API.

Two first-party clients are published and ready to use:

  • Jellyfin plugin: Bazarr+ Subtitles — subtitle search and download from inside Jellyfin 10.11+, served by your Bazarr+ providers.
  • VLSub Bazarr+ — VLC 3.0+ Lua extension. Hash-based and title-based search inside VLC; downloads load straight into the current playback.
What it is not. External Integration is not a way to share your library publicly. The endpoint is bearer-token authenticated, rate-limited per token, and SSRF-guarded. Only clients that have your token can search.

Enabling the endpoint in Bazarr+

  1. In Bazarr+, go to Settings > External Integration.
  2. Toggle Enable External Integration on. Bazarr+ generates the JWT and HMAC secrets on first enable.
  3. A prominent restart-required banner will appear. Click Restart Bazarr+.
  4. After the restart finishes, return to the same page. Copy the API Token shown in the panel. You will paste this into the Jellyfin plugin and the VLSub configuration.
Treat the token like a password. Anyone who has the token can search and download via your Bazarr+ instance. Don't paste it into chat threads, public configs, or screenshots. Rotate it (next section) if you ever suspect it leaked.

Rotating or revoking the token

On the same Settings page, the Regenerate button mints a fresh token and revokes the previous one immediately. Old clients will start getting 401s and need the new token pasted in. Use this when:

  • The token has been exposed (committed to a repo, posted in a screenshot, sent over an unencrypted channel).
  • You are decommissioning a client and want to make sure it can't talk to Bazarr+ any more.
  • You just want to rotate periodically as good hygiene.

The signed download tokens (which are minted per /compat/download and used to fetch subtitle bytes) are tied to the JWT they were issued under. Regenerating invalidates outstanding download URLs as well.

Installing the Jellyfin plugin

The recommended path is the plugin repository. Jellyfin can install and update from a manifest URL.

  1. In Jellyfin, go to Dashboard > Plugins > Repositories and click the + button.
  2. Add a repository:
    • Name: Bazarr+ Subtitles
    • URL: https://LavX.github.io/jellyfin-plugin-bazarr-plus/manifest.json
  3. Open the Catalog tab. Under Subtitles, find Bazarr+ Subtitles and click Install.
  4. Restart Jellyfin when prompted.
  5. Go to Dashboard > Plugins > My Plugins > Bazarr+ Subtitles > Settings.
  6. Fill in:
    • Bazarr+ URL: e.g. https://bazarr.example.com or http://bazarr.local:6767.
    • Bazarr+ API Token: paste the token from Bazarr+'s External Integration page.
  7. Save.
  8. Enable the provider per library at Dashboard > Libraries > [your library] > Subtitle Downloaders.
  9. Scan a library or click Find Subtitles on an item to verify end to end.

Manual install (downloading the release ZIP and dropping it into Jellyfin's plugins/ directory) and build-from-source instructions are in the plugin README.

Installing VLSub Bazarr+

VLSub Bazarr+ is a single Lua extension file plus a small library tree. The provided installer scripts handle both native VLC installs and Flatpak/Snap setups across Linux, macOS, and Windows.

Linux and macOS

Shell 
curl -sSL https://raw.githubusercontent.com/LavX/vlsub-bazarr-plus/main/scripts/install.sh | bash

Windows

PowerShell 
iwr -useb https://raw.githubusercontent.com/LavX/vlsub-bazarr-plus/main/scripts/install.ps1 | iex

Configure VLSub

  1. Restart VLC after install so the new extension is picked up.
  2. In VLC, open View > VLSub Bazarr+.
  3. Click Config in the extension window.
  4. Fill in:
    • Bazarr+ URL: e.g. http://bazarr.local:6767.
    • API Token: paste the token from Bazarr+'s External Integration page.
    • Languages: pick up to three.
  5. Save and close. From now on, View > VLSub Bazarr+ while a video is playing searches by hash first, then by title, and downloads the chosen result next to the video file.

The full installer detects native, Flatpak, and Snap VLC installations and writes to the right location for each. Manual install steps are in the VLSub Bazarr+ README.

How the pipeline works

When a Jellyfin or VLSub client searches:

  1. The client calls /compat/search on Bazarr+ with an IMDB id (or a title + season/episode + language).
  2. Bazarr+ runs your configured providers in parallel, with a bounded thread pool and dogpile coalescing so identical concurrent searches share a single backend call.
  3. The compat layer normalizes provider responses into the OpenSubtitles wire format the client expects, with download counts, ratings, FPS, HD flag, hash-match indicator, and language metadata.
  4. When the user picks a result, the client calls /compat/download. Bazarr+ returns a one-shot HMAC-signed stream URL with a TTL.
  5. The client follows the stream URL. Bazarr+ validates the token and rate-limits per JWT before serving the bytes.

For TVDB-keyed providers like Gestdown, Bazarr+ hydrates season/episode and TVDB series id from an IMDB id so they "just work". The OMDB refiner (broken since the Python 3 migration upstream) is also revived in v2.2 and can be enabled with a free OMDB API key on the same Settings page.

Security notes

  • Bearer-token auth with jti revocation. Logout immediately revokes the active JWT.
  • Rate limited per token with a sliding window on /compat/download, so a misbehaving client cannot eat the whole pool.
  • Signed download tokens with a TTL. The client never sees raw provider URLs.
  • SSRF guard with DNS rebinding protection on every outbound URL the compat layer touches. Loopback, RFC1918, and link-local addresses are blocked, and any IP that fails revalidation post-DNS resolution is refused.
  • Encryption at rest for the External Integration admin token and every other sensitive credential (provider keys, Sonarr/Radarr keys, Plex token, OpenRouter key). See the v2.2 release notes for the full list.

Troubleshooting

Jellyfin shows "Bazarr+ Subtitles" in the catalog as 404. The plugin manifest is published from the plugin repo. If you see a 404, no plugin release has been published yet for your Jellyfin version. Use the manual install path in the plugin README.

Plugin returns 401 from Bazarr+. The API token is wrong or has been rotated. Copy the current token from Bazarr+ > Settings > External Integration and paste it again into the plugin settings.

VLSub finds nothing for hash searches. Hash-based search only works if at least one of your enabled providers supports the file's hash. Try a title-based search as a fallback (VLSub auto-falls back when hash search returns empty).

"External Integration" toggle is missing in Settings. You're on Bazarr+ < v2.2.0. Pull ghcr.io/lavx/bazarr:v2.2.0 or :latest and restart.
