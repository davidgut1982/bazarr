Provider Hub v2.3

Bazarr+ v2.3.0 (Keystone) introduces Provider Hub: a plugin foundation for subtitle providers. It lets you install providers from catalog sources while Bazarr+ keeps third-party code staged, verified, and isolated.

What Provider Hub adds

  • Marketplace: browse installable subtitle provider plugins from Settings > Providers.
  • Catalog sources: use the official catalog source or add community GitHub catalog sources.
  • Trust labels: see whether a provider came from the official source or an untrusted custom source.
  • Staged lifecycle: installs, updates, and active removals wait for restart before they become live.
  • Worker validation: plugin code is imported and smoke-tested in a worker before Bazarr+ stages it.
  • Activity log: catalog refreshes, installs, updates, tests, removals, durations, versions, and errors are visible to operators.

Installing a provider

  1. Open Settings > Providers.
  2. Open the Marketplace tab.
  3. Refresh the official catalog source if the list is empty or stale.
  4. Review the provider card, including source, trust label, version, supported media, and languages.
  5. Click Install. Bazarr+ downloads the pinned bundle, verifies hashes, installs dependencies into an isolated environment, and validates the worker.
  6. Restart Bazarr+ when the staged-change banner appears.
  7. After restart, add the installed plugin to the enabled provider list and save settings.
Existing providers keep working. Provider Hub is additive. Shipped providers and existing provider settings continue to work after upgrade.

Sources and trust

Bazarr+ ships with an official Provider Hub catalog source. Community sources can be added as GitHub catalog URLs, but Bazarr+ stores custom sources as untrusted even if a caller tries to mark them trusted.

That does not block community plugins. It keeps the trust signal honest so operators can tell official catalog entries from custom source entries before installing code.

Restart lifecycle

Provider Hub does not hot-reload Python provider modules. Live unload and reload of provider code is risky when third-party dependencies are involved. Bazarr+ keeps the currently active provider running until restart, then promotes or removes staged plugins during startup.

If you see a pending restart banner, the change is staged correctly. Restart Bazarr+ to make it active.

Catalog and SDK

The official catalog lives at LavX/bazarr-provider-catalog. It contains the catalog manifest, provider packages, SDK tooling, and tests for provider authors.

The SDK is authoring tooling. Installed providers do not import it at runtime.

Catalog SDK 
python3 -B -m sdk validate
python3 -B -m sdk hash providers/<id>
python3 -B -m sdk build-catalog
python3 -B -m sdk smoke-test

Provider packages must declare a manifest, pinned files, SHA-256 hashes, bundle hash, entry module, entry class, supported media, languages, and exact hash-checked dependencies.

First catalog provider

SubtitleCat is the first real Provider Hub catalog provider. It targets the official catalog and supports movie and episode search without a login, broad language coverage, optional cached machine-translated subtitles, request delay control, and source page links in provider payloads.

Troubleshooting

Marketplace is empty. Refresh the official source under Settings > Providers. If you use a custom source, confirm the GitHub URL and branch are reachable.

Provider installed but does not run. Check for the pending restart banner. Active providers are promoted during startup, not while Python provider code is already loaded.

Install failed. Open the Activity tab. Hash mismatches, missing files, dependency failures, worker import errors, and smoke-test failures are recorded there.

Custom source looks untrusted. That is expected. Custom catalog sources are intentionally labeled untrusted so operators can distinguish them from the official source.
